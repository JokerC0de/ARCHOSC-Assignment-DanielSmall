Section 2: Adding Lab Content

I added the text from my working document into this new webpage using paragraph tags. I also added screenshots from my Pictures folder using image tags.

The paragraph tag is used to display written explanations, and the image tag is used to display screenshots as evidence of the work completed.

Section 3: GitHub Actions and YAML

I created a GitHub Actions workflow file called deployment.yml inside the .github/workflows folder. This workflow is used to automatically deploy the website when changes are pushed to the main branch.

The workflow successfully ran in the Actions tab, which shows that the deployment process worked correctly.

Section 4: Linking the Pages Together

I joined my pages together by adding a navigation bar at the top of each HTML page. The navigation bar uses hyperlink tags to connect index.html and github-cicd-2.html. 

<nav>
    <a href="index.html">GitHub & CI/CD Lab 1</a>
    <a href="github-cicd-2.html">GitHub & CI/CD Lab 2</a>
</nav>

The href attribute tells the browser which page to open when the user clicks the link. This makes the portfolio easier to use because the user can move between lab pages without manually opening separate files.
